# Next 웹툰 필터 설계

## 1. 목표와 범위

NTK Webtoon Source ID `260713001`의 Next 계열에 작품 목록 필터를 추가한다. 제목이 입력된 경우 기존 제목 검색을 우선하고, 제목이 비어 있을 때만 필터 전용 `/api/works` 요청을 사용한다.

이번 범위는 작품 구분, 주요 장르, 상세 장르, 플랫폼, 정렬과 필터 결과 페이지 이동이다. 요일 필터, 작품 상세, 회차 목록, 뷰어, Legacy 필터 변경은 포함하지 않는다. Mangayomi 클라이언트는 수정하지 않는다.

## 2. 조사 근거

2026-07-13 현재 `https://sbxh9.com/ing`, `https://sbxh9.com/end`와 배포된 Next 스크립트를 직접 확인했다. 사이트는 주소창 경로와 내부 목록 API를 분리한다.

| 작품 구분 | 화면 경로 | 내부 API 조건 |
|---|---|---|
| 전체 | `/ing?cat=all` | `status=ongoing&cat=all` |
| 일반 | `/ing` | `status=ongoing&cat=normal` |
| 비엘 | `/ing?cat=bl` | `status=ongoing&cat=bl` |
| 성인 | `/ing?cat=adult` | `status=ongoing&cat=adult` |
| 완결 | `/end` | `status=completed`, `cat` 생략 |

필터 변경 시 사이트는 `/api/works`를 호출한다. 응답은 `works`, `page`, `hasMore`, `pageSize`, `total`을 포함한다. 작품에는 `sourceWorkId`, `title`, `thumbnailUrl` 등이 있다.

전체 태그는 113개다. 사이트의 현재 메인 장르 판정 로직에 실제 태그 이름을 대조하면 주요 장르 13개, 상세 장르 100개로 나뉜다. 과거에 추정했던 21/92 분류는 사용하지 않는다.

## 3. Mangayomi 필터 UI

공식 확장 모델만 사용한다.

1. `SelectFilter` 작품 구분
   - 전체(기본값), 일반, 비엘, 성인, 완결
2. `HeaderFilter`
   - `장르: 체크=포함, 가로선=제외`
3. `GroupFilter` 주요 장르
   - 각 자식은 `TriState`
4. `GroupFilter` 상세 장르
   - 각 자식은 `TriState`
5. `SelectFilter` 플랫폼
6. `SelectFilter` 정렬

`TriState.state`는 `0=미선택`, `1=포함`, `2=제외`로 해석한다. 주요 장르와 상세 장르의 포함 ID는 하나의 `tag` 목록으로 합치고, 제외 ID는 하나의 `xtag` 목록으로 합친다. UI 그룹은 분리하지만 서버 조건은 동일한 태그 축이다.

## 6. 요청 직렬화

필터 요청은 항상 다음 값을 포함한다.

- `status`
- `withTotal=1`
- `page`
- `pageSize=42`

조건이 있을 때만 `cat`, `tag`, `xtag`, `plat`, `sort`를 추가한다. `sort=new`과 플랫폼 전체는 생략할 수 있다. 포함·제외 태그는 화면 표시 순서대로 중복 없이 쉼표로 결합한다.

완결 선택 시 `status=completed`를 보내고 `cat`을 생략한다. 다른 네 작품 구분은 `status=ongoing`과 해당 `cat` 값을 보낸다. 요일 관련 `day`, `xday`는 생성하지 않는다.

## 7. 검색 분기와 응답 변환

`search(query, page, filters)`의 Next 분기는 다음과 같이 동작한다.

1. `query.trim()`이 비어 있지 않으면 기존 `/search?q=...&field=title&match=contains` 제목 검색을 사용한다. 필터는 무시하며 검색 2페이지부터는 기존처럼 빈 결과를 반환한다.
2. 제목이 비어 있으면 `/api/works` 필터 목록을 요청한다.

각 API 작품은 다음과 같이 변환한다.

- `name = title`
- `link = /webtoon/{sourceWorkId}`
- `imageUrl = thumbnailUrl`을 현재 Next 호스트 기준 절대 URL로 정규화
- `hasNextPage = hasMore`

정상 빈 응답은 `works=[]`, `hasMore=false`다.

## 8. 오류 처리

- HTTP 200~399가 아니면 오류를 발생시킨다.
- `Content-Type`이 있으면 `application/json`이어야 한다.
- JSON 파싱 실패는 구조 오류로 보고한다.
- `works`가 배열이 아니거나 `hasMore`가 불리언이 아니면 구조 오류로 보고한다.
- 작품의 `sourceWorkId` 또는 `title`이 없으면 해당 응답을 조용히 축소하지 않고 구조 오류로 보고한다.
- 정상 빈 결과와 HTTP/JSON/구조 오류를 구분한다.
- Popular, Latest, 제목 검색, Legacy 파서는 변경하거나 공유하지 않는다.

## 9. 테스트와 출시

자동 테스트는 다음을 검증한다.

- 필터 순서와 공식 Mangayomi 타입
- 작품 구분 5개와 정확한 API 매핑
- 요일 필터 부재
- 주요 13개, 상세 100개의 정확한 ID와 대표 이름
- 복수 포함·제외 태그 결합
- 플랫폼과 정렬 매핑
- 페이지 번호와 `hasMore`
- 제목 검색 우선
- 정상 목록, 정상 빈 목록, HTTP 오류, 비 JSON, 잘못된 JSON, 구조 오류
- 기존 Popular, Latest, 제목 검색, Legacy 회귀 테스트
- `index.json`과 내장 `mangayomiSources` 일치

필터 프로토타입 버전은 `0.104`로 올린다. Webtoon 전체 완성 전까지 `0.2`로 올리지 않는다. `index.json` 메모에는 Next Popular, Latest, 제목 검색, 필터가 구현되었고 상세와 리더는 아직 미구현임을 명시한다.

## 10. 완료 기준

- Mangayomi에서 제목이 없는 검색 화면에 필터가 표시된다.
- 작품 구분, 장르 포함·제외, 플랫폼, 정렬 조합이 목록과 페이지 이동에 반영된다.
- 제목이 있으면 필터와 무관하게 제목 검색 결과가 나온다.
- 썸네일은 플랫폼 로고가 아니라 API의 `thumbnailUrl`을 사용한다.
- 전체 자동 테스트와 라이브 API smoke test가 통과한다.
