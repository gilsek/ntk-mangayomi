## 7. 필터 초기화와 페이지 초기화

- 각 SelectFilter의 첫 값은 전체이며 내부 값은 빈 문자열이다.
- 빈 필터 값은 쿼리에서 생략한다.
- 작가 입력 초기화는 `author`를 생략한다.
- 기본 정렬은 `as_update`, 방향은 `desc`다.
- 현재 사이트 필터 폼에는 `page` 입력이 없다.
- 사이트 UI에서 필터를 새로 선택하면 GET 폼 제출로 1페이지에서 다시 시작한다.
- Mangayomi에서도 새로운 검색 또는 필터 상태에는 `page=1`을 사용한다.
- 같은 상태에서 추가 페이지를 불러올 때만 전달받은 `page`를 추가한다.

기본 요청과 기본 필터값 요청은 작품 96개와 작품 URL 집합이 동일했다.

## 8. 요청 URL 및 리다이렉트

- 확장은 사람이 읽을 수 있는 표준 GET 쿼리를 생성한다.
- 서버는 필터 쿼리를 `302`로 `/webtoon/__q/{encodedQuery}`에 이동시킨다.
- HTTP 클라이언트의 정상 리다이렉트 처리를 사용한다.
- 확장에서 `/__q/` 값을 직접 인코딩하거나 생성하지 않는다.
- 리다이렉트 후 URL을 작품 URL이나 설정값으로 저장하지 않는다.

## 10. 페이지 계약

- 페이지 파라미터: `page`
- 기본 페이지: 1
- 목록 크기 96을 코드에 고정하지 않는다.
- 확인 결과:
  - 기본 1페이지: 96개
  - 2페이지: 96개
  - 기본 마지막 80페이지: 31개
- 현재 페이지: `li.active > a`의 숫자
- 페이지네이션 링크의 `page` 값 중 최대값을 마지막 페이지 후보로 사용한다.
- `hasNextPage = currentPage < maxPageFromPaginationLinks`
- 페이지 링크가 하나도 없으면 최대 페이지를 현재 페이지로 본다.
- 오른쪽 화살표는 다음 페이지가 아니라 다음 페이지 묶음이므로 `hasNextPage` 판정에 사용하지 않는다.

- 기본 80페이지: 현재 80, 최대 80, 다음 페이지 없음
- 3종 조합 1페이지: 현재 1, 최대 2, 다음 페이지 있음
- 3종 조합 2페이지: 현재 2, 최대 2, 다음 페이지 없음

## 11. 정상 빈 결과와 오류 구분

### 정상 빈 결과

- 상태 `200`
- `div.wr-none` 존재
- 텍스트 `등록된 작품이 없습니다.`
- 빈 목록 반환
- `hasNextPage=false`

### 구조 오류

다음 조건이면 빈 결과로 숨기지 않고 구조 변경 오류로 처리한다.

- 상태는 `200`이지만 `#webtoon-list-all`과 `div.wr-none`이 모두 없음
- 목록 행은 있으나 제목 또는 작품 상대 경로가 없음
- 응답이 HTML이 아님
- 로그인, 유지보수 또는 보안 확인 페이지로 판단되는 별도 표지가 있음

### HTTP 오류

- 3xx는 정상 리다이렉트 범위에서 클라이언트가 처리한다.
- 최종 4xx와 5xx는 목록 없음으로 변환하지 않는다.
- 타임아웃과 네트워크 오류도 빈 목록으로 변환하지 않는다.
- 오류 시 다른 parserFamily로 자동 전환하지 않는다.

## 12. 최소 fixture 후보

구현 승인 후 현재 응답에서 다음 최소 구조만 별도 fixture로 작성한다.

1. 숫자 작품 키가 있는 정상 행
2. slug 작품 키가 있는 정상 행
3. 표지 이미지가 없는 정상 행
4. 첫 페이지 페이지네이션
5. 마지막 페이지 페이지네이션
6. 정상 빈 결과 `.wr-none`
7. 필터 폼과 전체 필터 값
8. 목록 컨테이너가 없는 구조 오류 HTML

fixture에는 광고, 전체 페이지, Cookie, 토큰, 불필요한 스크립트와 스타일을 포함하지 않는다.

## 13. 승인된 구현 계약

2026-07-13에 다음 항목을 승인했다.

1. `getPopular`을 조회순 `as_view`로 매핑한다.
2. 한 글자 제목 검색은 요청하지 않고 빈 결과를 반환한다.
3. 작가를 `TextFilter`로 제공한다.
4. 분류, 요일, 초성, 플랫폼, 장르와 정렬을 `SelectFilter`로 제공한다.
5. 성인웹툰이 주요 사용 목적이므로 Source를 `isNsfw: true`로 등록한다.
6. 표지 누락 작품을 빈 `imageUrl`로 유지한다.
7. 현재 페이지와 페이지 링크 최대값을 비교해 `hasNextPage`를 결정한다.
8. 신규 Source ID `260713001`을 사용한다.

이 승인은 Legacy Webtoon 작품 목록의 상세 구현 계획 작성까지 허용한다. JavaScript 구현 파일, 테스트 코드와 `index.json`을 실제로 생성·수정하기 직전에는 변경 파일과 검증 방법을 다시 제시하고 사용자 승인을 받는다.
